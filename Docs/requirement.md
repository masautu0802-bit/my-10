# 要件定義書 — マイ10

## 1. プロジェクト概要

**サービス名**：マイ10
**背景・目的**
「なんか欲しいけど何が欲しいかわからない」というモヤモヤを解消するキュレーション型 Webサービス。
ユーザーは他ユーザーのセレクトショップを閲覧し、価値観や趣味の近い人のおすすめ商品を見つけることができる。

**リリース目標**
初期リリースを **2026年 夏** までに実施。

---

## 2. 想定ユーザー / ペルソナ

**メインターゲット**
- 年齢：30〜40代女性
- 利用環境：通勤中のスマホ閲覧を想定（平均20分）だが、PC・タブレットでの利用も対応予定。

**ペルソナ例**
- 35歳、女性
- ヨガやピラティスに興味あり
- 何を買えば良いか分からず、他ユーザーおすすめを参考にしたい

---

## 3. ステークホルダー

| 役割 | 関係 |
|------|------|
| プロダクトオーナー | 企画・優先度決定 |
| 開発者 | 実装 |
| ユーザー | 利用者 |
| 将来のコミュニティ | 相互作用・レビュー機能（後実装） |

---

## 4. 機能要件（優先度）

| ID  | 機能 | 説明 | 優先 |
|-----|------|------|------|
| F1 | ユーザー登録/ログイン | Supabase Auth による認証 | MUST |
| F2 | プロフィール編集 | 興味・好み設定 | SHOULD |
| F3 | セレクトショップ一覧 | 他ユーザーおすすめ一覧 | MUST |
| F4 | 商品詳細表示 | 商品詳細・説明 | SHOULD |
| F5 | ECリンク | 外部購入リンク | MUST |
| F6 | フォロー/お気に入り | 好みのユーザー追跡 | SHOULD |
| F7 | レコメンド | 趣味が似たユーザー提示 | COULD |
| F8 | コメント/レビュー | ユーザー間相互作用（将来） | COULD |

---

## 5. 非機能要件

- **モバイル最適化**：モバイル UX を最優先
- **レスポンス性能**：主要ページは可能な限り 2〜3 秒以内
- **セキュリティ**：HTTPS、セッション管理、脆弱性対策
- **拡張性**：機能追加に柔軟に対応可能な構造
- **信頼性**：高可用性・安定運用を視野に設計

## 8. ユーザーストーリー（例）

- ユーザーとして、興味あるセレクトショップを一覧で見たい
- ユーザーとして、商品詳細を確認し外部サイトで購入したい
- ユーザーとして、好みが似たユーザーをフォローしたい
